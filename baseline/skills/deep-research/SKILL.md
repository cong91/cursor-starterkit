# Deep Research

## Source hierarchy

1. Official docs, type definitions, source code (Tier 1)
2. Official blogs, changelogs, specs (Tier 2)
3. Browser compat, release notes (Tier 3)
4. Stack Overflow, blogs, AI docs (Tier 4 — verify before citing)

## Process

1. State what to verify
2. WebSearch Tier 1-2 first; use Context7 MCP for library docs
3. Cross-check conflicting claims; state conflicts explicitly
4. Summarize with links; label inferences as inferences

## Rules

- For real products (SDKs, devices, APIs), search current state — never guess versions or launch dates
- Cite URLs for non-obvious claims
- Distinguish source-backed facts from creative interpretation
