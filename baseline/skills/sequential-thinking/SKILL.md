# Sequential Thinking

## When to use

- A problem has many interdependent parts
- You need to revise hypotheses mid-reasoning
- Planning has branches or dead-ends worth exploring

## How

Call the `sequential-thinking` MCP tool with your current thought; it supports
revision, branching, and a running hypothesis. Use it to externalize reasoning
before committing to an implementation.

## When NOT to use

- Simple, single-step tasks
- Tasks where the answer is obvious from one read of the code
